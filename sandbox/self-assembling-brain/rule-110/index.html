Rule 110: Elementary Cellular Automaton

Rule 110 represents a profound intersection of simplicity and complexity in computational systems. Unlike Conway's Game of Life, which operates on a 2D grid with multiple neighbors, Rule 110 works on just a 1D line where each cell considers only itself and its two adjacent neighbors. Yet this deceptively simple system achieves something remarkable: it's proven to be Turing complete, meaning it can simulate any computer program given enough time and space.

This cellular automaton perfectly illustrates how complex behaviors can emerge from simple rules—a key principle in understanding both neural development and artificial intelligence. Like neurons following local chemical signals to form elaborate networks, Rule 110's cells create intricate patterns through purely local interactions. This bottom-up emergence mirrors how our brains develop: from simple cellular rules to complex cognitive architectures.

How it works

Pattern Examples

Rule 110 generates several distinctive patterns that showcase different aspects of emergent computation. The "Ether" pattern forms a stable background, while "Gliders" transport information across the grid. These patterns can combine to perform logical operations, forming the basis of its computational universality.

The Algorithm

At its core, Rule 110 operates on a beautifully simple principle: each cell's fate is determined by just three bits of information—its own state and its two neighbors'. This creates eight possible patterns, each mapping to a single output bit: 


// The core algorithm that determines the next state
function computeNextGeneration(currentRow) {
  const nextRow = new Array(currentRow.length).fill(0);
  
  for (let i = 0; i < currentRow.length; i++) {
    // Get the three-cell neighborhood
    const left = i === 0 ? 0 : currentRow[i - 1];
    const center = currentRow[i];
    const right = i === currentRow.length - 1 ? 0 : currentRow[i + 1];
    
    // Convert neighborhood to pattern index
    const pattern = (left << 2) | (center << 1) | right;
    
    // Apply Rule 110 lookup
    nextRow[i] = RULE_110[pattern];
  }
  return nextRow;
}

The name "Rule 110" comes from the binary number 01101110 (decimal 110) formed by reading the output bits in reverse. This simple lookup table, combined with the sequential application across a row of cells, creates complex patterns that can encode arbitrary computations. 


// The pattern definitions that create Rule 110
const RULE_110 = [
  0, // 000 -> 0  (No neighbors alive)
  1, // 001 -> 1  (Right neighbor only)
  1, // 010 -> 1  (Center only)
  1, // 011 -> 1  (Center and right)
  0, // 100 -> 0  (Left only)
  1, // 101 -> 1  (Left and right)
  1, // 110 -> 1  (Left and center)
  0  // 111 -> 0  (All alive)
];

These eight rules create a delicate balance between growth and decay. Unlike simpler cellular automata that quickly stabilize or dissolve into chaos, Rule 110 maintains a complex edge between order and randomness—much like the neural networks in our brains.

How it works

Click cells in the top row to set their initial states. Press Play to watch the pattern evolve downward, or use Step to advance one generation at a time. Use Randomize to create a random starting pattern, or Clear to start fresh.

The Rules

Each cell follows Rule 110's pattern matching:

  • Three-Cell Window: Each cell considers itself and its two neighbors
  • Eight Patterns: The three cells create 8 possible binary patterns
  • Binary Output: Each pattern maps to either 0 (dead) or 1 (alive)
  • Sequential Update: All cells update simultaneously based on the current row

Controls

  • Play/Pause: Start or stop the evolution
  • Step Forward: Advance one row at a time
  • Randomize: Create a random initial pattern
  • Clear: Reset to an empty state

Notable Patterns

Watch for these emergent structures:

  • Ether: The background pattern that repeats every 7 steps
  • Gliders: Small patterns that move diagonally across the grid
  • Collisions: Interactions between gliders that can perform computation
